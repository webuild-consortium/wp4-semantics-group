# European business wallet vocabulary

The EBWV consists of three layer:

* Terminology - Abstract concepts of all terms used in the vocabulary including their realtionships. The Terminology is define using [SKOS](https://www.w3.org/2004/02/skos/)
* Vocabulary - Formal definition of the classes, properties, individuals and datatypes of the semantic model using [OWL](https://www.w3.org/TR/owl-ref/)
* Context - JSON-LD is used to maps the terms used in the credentials to the semantic definitions in the vocabulary [JSONLD](https://www.w3.org/TR/json-ld11/)

The terminology is created using the nordic interoperability platform and is available at:\
https://w3id.org/ebwv/terminology

The latest version of the European Business Vocabulary is available at:\
https://w3id.org/ebwv \
the following formats are selectable by specifying the mime type: html (default), turtle, jsonld

The corresponding JSONLD context for the semantic mapping is available at:\
https://w3id.org/ebwv/v0.1



## Folder Structure

The folder `vocab` contains sources and scripts to generate all artifacts of the semantic model description.

The folder `docs` contains all built artifacts. The artifacts are published using github pages:\
https://webuild-consortium.github.io/wp4-semantics-group/ebwv

```

├── vocab
│   ├── scripts
│   └── src
│       ├── vocabulary.yml          // vocab definition in yml
│       ├── template.html           // template for html representation of the vocabulary
│       └── class-diagram.mermaid   // part of html representation
└── docs
    ├── _config.yml // config for jekyll-redirect-from
    └── ebwv        // European Business Wallet Vocabulary
        ├── terminology.ttl     // terminology in turtle format
        ├── terminology.jsonld  // terminology in jsonld format
        ├── vocabulary.html     // vocabulary in html format (single page)
        ├── class-diagram.svg   // part of single page
        ├── vocabulary.turtle   // vocabulary in turtle format
        ├── vocabulary.jsonld   // vocabulary in jsonld format
        └── v0.1    // context version
            └── vocabulary.context.jsonld   // jsonld context
        
```

The tool yml2vocab is used to generate the vocabulary in html, turtle and jsonld format as well the jsonld context.

The tool mmdc is used to generate a svg file of the class diagram for the html documentation.



The turtle and jsonld format of the terminology are generated by the platform and downloaded to github.

## Build Artifacts

In order to build all artifacts and download the terminology from the nordic interoperability platform, execute the build script in the root folder of the project:

```console
wp4-semantics-group % vocab/scripts/build.zsh
```
